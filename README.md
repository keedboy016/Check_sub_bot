# тг бот для раздачи сборок

бот проверяет подписку на канал и если всё ок — даёт доступ к меню со сборками (большие зип файлы)

---

## что умеет

- проверяет подписку на @pweper (обязательный) + доп канал из ссылки
- меню сборок с описанием которое ты сам вписываешь
- большие файлы без ограничений — telethon принимает, pyrogram отдаёт
- система сборщиков — можешь дать людям доступ чтобы они заливали свои сборки и делали ссылки на свои каналы
- полная статистика в админке
- всё хранится в json, ничего лишнего

---

## файлы

## команды

| команда | кто | что делает |
|---|---|---|
| /start | все | начало, проверка подписки |
| /id | все | показывает твой тг айди |
| /admin | только ты | открывает админку |
| /builder | сборщики | их меню |

---

## как пользоваться админкой

пишешь /admin, дальше всё через кнопки

**группы/ссылки** — создаёшь группу, указываешь ключ + канал, получаешь ссылку вида `t.me/бот?start=ключ`. по этой ссылке бот будет требовать подписку на @pweper и на канал из группы

**сборки** — добавляешь сборку: ключ → описание → кидаешь зип. описание это то что увидят пользователи в меню кнопкой

**сборщики** — добавляешь человека по его тг айди (он может узнать его через /id). после этого у него открывается /builder

**статистика** — всё подробно: юзеры, активность, скачивания по каждой сборке

---

## меню сборщика

сборщик через /builder может:
- выбрать сборку из списка → указать свой канал → получить ссылку. по ней пользователи должны подписаться на @pweper + его канал
- залить свою сборку — вводит описание, кидает зип, сборка попадает в общий список

---

## важно

- бот должен быть **администратором** в каналах иначе не сможет проверять подписку
- `tgcrypto` в requirements обязателен, без него pyrogram будет ругаться
- при первом запуске создадутся два файла сессий `pyro_bot.session` и `tele_bot.session` — это нормально, не удалять
